<Section title="Redesign brief — structure and requirements" icon="📝" id="brief">

### What a redesign brief must include

1. **What is being redesigned:** the specific component or feature (e.g. the cap, not 'the whole bottle')
2. **Who the user is:** specific user profile (e.g. users with limited grip strength, children under 12, outdoor sports users)
3. **What the original weakness is:** referenced to the M1 analysis (e.g. 'the existing cap requires 8 turns to open')
4. **What the improvement must achieve:** measurable target (e.g. 'the redesigned cap must open in ≤ 1 rotation')

### Example redesign brief

"I am redesigning the cap of a 500 ml PET water bottle to improve one-handed use for users with limited grip, based on analysis in the M1 package showing the existing cap requires 8 turns to open — a barrier for users with grip strength below 30 N. The redesigned cap must open with a single 90-degree turn."

### Section B planning — worked example

**Scenario:** 500,000 PP food storage lids per year.

**Sub-parts (read all before writing):**
- (a) Name the commercial process — "injection moulding" [plan: 1 word]
- (b) Describe how it works — [plan: hopper / melt / inject / cool / eject — 5 points]
- (c) Give one advantage — [plan: repeatability → identical units → low rejection rate]
- (d) State one QC check — [plan: thread diameter go/no-go gauge; tolerance ±0.1 mm]

**Plan time: 2 minutes. Write time: 8 minutes.** Total: 10 minutes for a 4-sub-part question.

</Section>

<WorkedExample title="Specification point — traceable to M1">

**M1 finding:** "The cap requires 8 turns to open — tested and recorded. Users with reduced grip strength find this fatiguing."

**Specification point derived from this finding:**

"The redesigned cap must open fully in ≤ 1 rotation of 90° or less because the M1 analysis showed the existing cap requires 8 full rotations, which is a barrier for the target user group (grip strength below 30 N)."

**What makes this traceable:** it names the original test result (8 rotations), names the affected user, and sets a specific measurable target (≤ 1 rotation of ≤ 90°). An examiner can verify that the specification point comes from the analysis.
</WorkedExample>

<Activity title="Redesign brief + specification + sketches + Section B practice" time="90 min" grouping="Individual (drawing colleague circulates for sketch feedback)" materials={["M1 package returned (with teacher comments)", "A4 folder pages", "Paper 3 Section B practice question sheet"]}>

**Redesign brief + specification (0–35 min):**
1. Write the redesign brief: 2–3 sentences including original product, user, M1 finding, and improvement target.
2. Draft the specification: at least 5 points, each measurable and justified, each traceable to an M1 finding. Include one sustainability point.

**Design sketches (35–70 min):**
3. Produce two annotated design sketches of the redesigned component — each showing a different approach (two different mechanisms or forms).
4. Drawing colleague circulates: accepts or returns each sketch for more annotation. Incorporate at least one piece of feedback before the session ends.

**Paper 3 Section B practice (70–90 min):**
5. Read the Section B scenario question. Write a 2-minute bullet plan in the margin.
6. Write the full sub-part answer (5 marks) for the given polymer manufacturing scenario.
7. Self-check: does every step have the material state at that step?
</Activity>

<Support title="Section B — how many marks per point?">

For a "describe how" sub-part worth 4 marks: write 4 numbered steps. For 5 marks: write 5 steps. One clear step = one mark. A step that combines two actions into one sentence often loses a mark. Separate: "The mould closes, pinching the parison at the base" and "Compressed air is injected, inflating the parison against the mould walls" — two sentences = two marks.
</Support>
