<KeyTerms terms={[
  { term: "Design evaluation", definition: "Comparing the redesigned component to each specification point with: the criterion stated, the test method named, a measured result given, and a honest PASS/FAIL/PARTIAL judgement. An evaluation that names only strengths is not a complete evaluation — the examiner requires evidence that weaknesses were identified and acknowledged." },
  { term: "Acknowledged trade-off", definition: "An honest statement that the redesign solves one problem but creates another, or that one specification point conflicts with another. Example: 'The bayonet-lock cap opens in 90°, meeting the one-handed use criterion. However, the deeper cap body required by the bayonet mechanism increases material mass by approximately 8%, partially conflicting with the sustainability specification point.' Acknowledging trade-offs demonstrates engineering thinking." },
  { term: "Section B question choice strategy", definition: "Paper 3 Section B offers two questions. <Section title="Evaluation structure" icon="📋" id="evaluation">

### Specification evaluation table

| Specification point | Test method | Result | PASS/FAIL |
|--------------------|------------|--------|-----------|
| Cap opens in ≤ 1 rotation | Rotate cap from fully closed; count rotations | 1 × 90° turn | PASS |
| Cap material is food-safe | Check material data sheet | PP food-grade specification confirmed | PASS |
| Cap opens one-handed | One-hand test: thumb + index only | Opened without repositioning grip | PASS |
| Cap mass ≤ current cap +10% | Weigh on digital balance | Original: 4.2 g; redesign: 4.7 g (+12%) | FAIL |
| Design uses ≤ same quantity of PP as original | Calculate volume from CAD model | Redesign volume 8% larger | PARTIAL |

### Comparison — redesign vs original (150 words minimum)

Structure: (1) what the redesign improved + evidence; (2) what the original did better + honest acknowledgement; (3) what a third iteration would change.

</Section>

<WorkedExample title="Evaluating the redesigned cap against three specification points">

**Point 1 — opens in ≤ 1 rotation:**
"Tested: rotate from closed to open — one 90° turn to disengage the bayonet lock. Result: 1 rotation. Specification: ≤ 1 rotation. **PASS.**"

**Point 2 — cap mass ≤ original + 10%:**
"Weighed on digital balance: original cap 4.2 g; redesigned cap 4.7 g (+12%). Specification: ≤ original + 10% = ≤ 4.6 g. Redesigned cap is 0.1 g above the limit. **FAIL.** This failure arises because the deeper cap body required by the bayonet mechanism adds approximately 0.5 g of PP. A third iteration could reduce the cap wall thickness from 2 mm to 1.5 mm to recover this mass without compromising the bayonet geometry."

**Point 3 — food-safe material:**
"Checked PP grade against material data sheet — food-grade PP approved for contact with potable water. **PASS.**"
</WorkedExample>

<Activity title="Evaluation + comparison + Section B timed practice" time="90 min" grouping="Individual" materials={["Design package from M2 (returned)", "Revision card from W8", "Paper 3 Section B question sheet (two questions)"]}>

**Evaluation (0–45 min):**
1. Write evaluation paragraphs for all five+ specification points using the criterion + test method + result + PASS/FAIL format.
2. Acknowledge at least one trade-off: one way the redesign is better AND one way it is worse than the original.

**Comparison paragraph (45–60 min):**
3. Write 150 words: what the redesign improved (with evidence), what the original did better (honest), what a third iteration would change and why.

**Package completeness check (60–70 min):**
4. Check the design package against the M2 completeness list. Note any element that was missing — write what you would include in a second submission.

**Section B timed practice (70–90 min):**
5. Read both questions (5 minutes). Decide which to answer based on your process knowledge. Do not choose based on which question you read first.
6. Bullet-plan each sub-part (2 minutes).
7. Write the full answer (18 minutes).
8. Self-mark using the mark scheme structure. Write one revision note.
</Activity>

<Support title="Section B strategy — how to read the question">

Read the whole question before writing a single word. Then ask: (1) Do I know the process name? (2) Can I write at least three steps of how it works? (3) Do I have a QC example? If all three yes → start here. If no → read the other question. You are choosing based on content knowledge, not on the topic name in the question header.
</Support>
