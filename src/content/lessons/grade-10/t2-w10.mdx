The product is complete. Evaluation is not a formality at the end — it is the step that closes the design loop. An evaluation that only names strengths is not an evaluation; it is a celebration. The examiner wants to see you identify what did not work and propose a specific improvement.

<KeyTerms terms={[
  { term: "Valid evaluation", definition: "An evaluation structured as: criterion stated → test method → measured result → judgement (PASS / FAIL / partially met). Example: 'The base measures 278 mm wide (steel rule); specification requires ≤280 mm. Within tolerance — PASS.' Evaluation without evidence earns lower marks." },
  { term: "Paper 1 part (f)", definition: "Choose a material for a component in a given product. Name at least two properties and link each to the design context. The answer must not use generic words — 'strong' alone earns nothing; 'high tensile strength, meaning the bracket resists the load without stretching' earns the mark." },
  { term: "Paper 1 part (g)", definition: "Evaluate a given design idea against stated specification criteria. Not 'I like it' but: 'This design meets / does not meet [criterion] because [evidence from the sketch or context].' The evidence comes from the brief or the design itself — you are the reviewer, not the designer." },

<Section title="Evaluation structure — minimum for a strong response" icon="📋" id="evaluation">

### Specification test table — 6+ points

| Specification point | Test method | Result | PASS / FAIL |
|--------------------|------------|--------|------------|
| Fits within 300 × 200 mm footprint | Steel rule measurement | 276 × 194 mm | PASS |
| Supports ≥ 2 kg without deflection | Load test with measured weights | 2.1 kg loaded, no visible sag | PASS |
| Uses ≥ 2 material families | Visual inspection of components | Pine (timber) + acrylic (polymer) | PASS |
| Acrylic panel is translucent | Hold against light source | Contents visible through panel | PASS |
| All edges smooth — no sharp points | Run gloved hand over all edges | Acrylic edge sharp at one corner | FAIL |
| FSC-certified timber used | Check supplier documentation | FSC stamp on pine off-cut bag | PASS |

### Written evaluation — 150-word minimum structure

1. **Introduction (20–30 words):** does the product overall meet the brief? (one honest sentence)
2. **Strengths with evidence (50–70 words):** two or three criteria that are passed, each with the test result
3. **Weaknesses with evidence (40–50 words):** one or two criteria that are failed or partially met, with the measured evidence
4. **Proposed improvements (20–30 words):** one specific, actionable improvement linked to each weakness

</Section>

<WorkedExample title="Paper 1 parts (f) and (g) answered">

**Setup:** The examiner provides a brief for a bicycle pannier bag bracket (mild steel, load-bearing).

**Part (f) — material justification:**

"3 mm mild steel is suitable for the bracket because:
1. It has high tensile strength, meaning it can support the weight of a loaded pannier (up to 10 kg) without stretching or bending.
2. It is ductile, allowing it to be bent to the required angle during manufacture without cracking."

**Part (g) — evaluating against criteria:**

"The brief states the bracket must attach to the bike frame without permanent modification. The shown design uses clamp bolts, which can be tightened and released without tools — this meets the criterion. However, the brief also states the bracket must support 10 kg. The design sketch shows a single bolt attachment point, which concentrates the load at one point — this may not meet the load criterion without additional support."
</WorkedExample>

<Activity title="Full evaluation + photography + Paper 1 (f)+(g)" time="90 min" grouping="Individual" materials={["Completed desk organizer", "Camera or phone", "White background paper/card", "Steel rule", "Measured weights (500 g, 1 kg)", "Paper 1 quick practice sheet"]}>

**Specification test table (0–40 min):**
1. Complete the test table for all 6+ specification points. Measure; do not estimate. Record actual numbers.

**Written evaluation (40–65 min):**
2. Write the 150+ word evaluation: introduction → strengths (with test evidence) → weaknesses (with evidence) → proposed improvements (specific).

**Photography (65–75 min):**
3. Set up a plain white background. Photograph: (a) front view, (b) side view, (c) detail of the most interesting construction feature.
4. Print or file the photographs with captions: "This photograph shows ___ and provides evidence of ___."

**Paper 1 (f)+(g) practice (75–90 min):**
5. Answer parts (f) and (g) for a fresh product context (teacher provides).
6. Peer-mark: each answer needs property name + function link for (f), and criterion + evidence for (g).
</Activity>

<Support title="If a specification point fails — write it honestly">

"The acrylic edge at the front right corner is sharp — this fails the safety specification point (no sharp edges). This occurred because the polishing sequence was cut short in Week 9. In a second iteration I would complete all stages of the acrylic edge polishing sequence (120 → 240 → 400 grit + metal polish) before assembly."

A specific honest failure with a named correction is higher quality evidence than a generic pass with no measurement.
</Support>
