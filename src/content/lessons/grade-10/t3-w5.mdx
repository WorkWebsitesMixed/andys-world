Analysis is only valuable if it leads somewhere. Today you close the M1 package by mapping the product as a system, testing it honestly against criteria, and identifying specific improvements you could design. The final act is choosing your redesign direction — the thread that connects today's analysis to Week 6's redesign.

<Callout type="exam" title="M1 package — complete and ready for W6 collection" id="graded">
  **M1 (20% of term) is collected at the start of Week 6.** Use the homework session after today to tidy the package. Everything goes in: parts list, ACCESS FM observations, two annotated drawings (W4), function map, test results table, improvement list, and three design direction sketches with chosen direction statement. Nothing can be added after collection.

<KeyTerms terms={[
  { term: "Function map", definition: "A diagram showing how the components of a product interact as a system. Written as: input → process → output, with each component's role mapped to one of these stages. For the water bottle: input (liquid is poured in) → process (body stores and protects, cap seals, gasket prevents leakage, grip texture prevents slipping) → output (liquid delivered to user without leaking)." },
  { term: "Testing against criteria", definition: "Structured testing using named criteria, test methods, and measured results. Format: criterion | test method | result | PASS/FAIL. The result must be measurable ('no leakage in 30 seconds of inversion', not 'it was fine'). Criteria come from what the product should do — its function, safety, ergonomics, and user needs." },
  { term: "Specific improvement", definition: "An improvement that is linked to test evidence and names a specific change. 'Make it more comfortable' is generic. 'The grip diameter of 72 mm is too wide for a smaller hand — reducing to a tapered waist at 60 mm midpoint would reduce grip span and improve ergonomics for users with smaller hands' is specific. Specific improvements lead directly to a redesign brief." },
  { term: "Design direction", definition: "A broad approach to redesigning the product — not a finished design, but a direction of travel. Each design direction addresses a different improvement from the list. Choosing a direction means committing to one improvement as the focus of the redesign and sketching three rough concepts that develop that direction." },
]} />

<Section title="Function map — how to build one" icon="🔧" id="function">

### The system view

A product is a system: inputs enter, processes transform them, outputs are delivered to the user.

**Water bottle system:**

| Stage | Components | Role |
|-------|-----------|------|
| Input | — | User fills bottle with liquid |
| Process | Body | Stores and contains the liquid |
| Process | Cap + thread | Seals the bottle when closed |
| Process | Gasket | Prevents leakage at the thread interface |
| Process | Grip texture | Allows user to hold the bottle securely |
| Process | Label | Communicates brand and nutritional information |
| Output | — | Liquid delivered to user via the open cap |

### Testing five criteria

| Criterion | Test method | Result (example) | PASS/FAIL |
|----------|------------|-----------------|-----------|
| No leakage when inverted | Fill, close cap, invert 30 s, hold over white paper | Minor drip at thread — wet spot on paper | FAIL |
| Opens with one hand | Open cap using thumb and index of one hand only | Required repositioning grip — not one-handed | FAIL |
| Stable when placed on flat surface | Place on desk, release — no support given | Remained upright — no tipping | PASS |
| Grip diameter suitable | Measure with calliper | 72 mm at widest point | Borderline |
| Cap opens in ≤ 3 turns | Count cap rotations from fully closed to open | 8 turns required | FAIL |

</Section>

<WorkedExample title="Specific improvement vs generic improvement">

**Generic:** "Make the cap easier to open."

**Specific (improvement linked to test evidence):** "The cap requires 8 turns to open fully — for users with reduced grip strength (below 30 N), repeated twisting is fatiguing. A bayonet-lock cap (90-degree quarter-turn) would open in one motion, eliminating the need for multiple rotations. Evidence: 8 turns measured in the testing session."

Notice: the specific version names the tested result (8 turns), names the affected user (reduced grip strength), and names the specific solution (bayonet-lock cap). It can be turned directly into a redesign brief.
</WorkedExample>

<Activity title="Function map + testing + improvement list + design directions" time="90 min" grouping="Individual" materials={["Water bottle from W4", "Calliper", "White paper", "Timer", "A4 folder pages"]}>

**Function map (0–40 min):**
1. Draw a function map of the water bottle: input → all process components with their roles → output. Label every component with its material and function.

**Testing exercise (40–65 min):**
2. Test the bottle against five criteria. Use the testing table format: criterion | test method | result | PASS/FAIL.
3. Be specific about the method and the result. Do not estimate — measure.

**Improvement list (65–75 min):**
4. List at least four specific improvements, each linked to a test result. Use the format: "The [tested measurement] shows that [problem]. A specific improvement would be [specific change], because [reason linked to evidence]."

**Design direction sketches (75–88 min):**
5. Sketch three design directions: one per key improvement. Each is a rough thumbnail — not a finished design.
6. Choose one direction. Write one sentence of criteria-based justification: "I chose Direction ___ because it addresses [the most impactful improvement identified in testing], which is supported by [specific test result]."

**Tidy for collection (88–90 min):**
7. Write the homework note: "For W6 collection: tidy the W4 drawings, type up the parts list and improvement list. Ready by Week 6 start."
</Activity>

<Support title="How to test the 'one-handed use' criterion without specialist equipment">

Define the test before starting: "I will attempt to open the cap using the thumb and index finger of my dominant hand only, without repositioning my grip on the bottle body." Test three times. Record: how many grip repositions were needed. If any repositioning was needed, the criterion is failed. This is repeatable — anyone else can run the same test and get a comparable result.
</Support>
