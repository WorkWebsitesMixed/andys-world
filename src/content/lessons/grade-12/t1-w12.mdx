The lab report is almost done. Today you review someone else's work with a structured checklist, receive their feedback on yours, make corrections, and submit your final PDF. The peer review checklist is designed so that a student who knows LaTeX — but not Chemistry at a deep level — can still give useful technical feedback.

<Callout type="tip" title="Why peer review?" id="why">
  A second pair of eyes finds errors you're blind to in your own work. Reviewing someone else's LaTeX also forces you to read the source code critically — and you'll spot in their document the same mistakes hiding in yours. The checklist prevents the review from being vague.
</Callout>

<Section title="The peer review checklist" icon="💻" id="checklist">

Share your Overleaf project with your partner (view-only access: Share → Anyone with the link → Viewer). Your partner uses this checklist to review your document. Write all feedback on a single sheet — section by section, not just a summary.

| # | Check | Pass / Fix needed |
|---|---|---|
| 1 | All chemical formulae use `\ce{}` — no plain-text subscripts visible in the PDF | |
| 2 | Every figure has a `\caption{}` and a `\label{}` | |
| 3 | Every figure is cross-referenced in the text (e.g. `Figure~\ref{fig:x}` in the body) | |
| 4 | Figure numbers in the text match the figure numbers in the PDF (cross-references not broken) | |
| 5 | Results table uses `booktabs` rules (`\toprule`/`\midrule`/`\bottomrule`) — no vertical lines | |
| 6 | All in-text citations use `\cite{}` — no manually typed [1] or (Author, 2020) | |
| 7 | Bibliography generated by `\printbibliography` — not typed manually | |
| 8 | No section is empty | |
| 9 | The PDF compiles without errors (no red error bar in Overleaf) | |
| 10 | No orphan headings (heading followed immediately by another heading) | |

For any item marked "Fix needed": write the section name and what specifically is wrong. "Figure 2 has no cross-reference in the text" is useful feedback. "Needs more LaTeX" is not.

</Section>

<Section title="Corrections phase" icon="💻" id="corrections">

You have 40 minutes to fix every item your reviewer flagged. Priority order:

1. **Compile errors** first — a document that doesn't compile cannot be submitted.
2. **Missing `\ce{}`** — replace all remaining plain-text formulae.
3. **Missing cross-references** — find every figure and table; add at least one `\ref{}` in the text for each.
4. **Bibliography issues** — confirm `\addbibresource{refs.bib}` is in the preamble and `\printbibliography` is before `\end{document}`.
5. **Empty sections** — even two sentences is better than nothing.

Both teachers circulate. Technical format questions → ICT teacher. Content questions → Chemistry teacher.

</Section>

<Section title="Export and submit" icon="💻" id="submission">

When your document compiles cleanly and all checklist items pass:

1. **Overleaf:** Menu (top-left) → Download → PDF.
2. **Rename the file:** `[YourSurname]_LabReport_ICT12.pdf` — exact format, no spaces.
3. **Submit** to the folder or platform your teacher specifies.
4. **Confirm the submitted file opens** — download it once from the submission folder and check the PDF renders correctly.

Do not submit the `.tex` file. The PDF is the deliverable.

</Section>

<WorkedExample title="Turning checklist feedback into a specific fix">
**Feedback received:** "Check 2 — Figure 2 has no label."

**Source code (before):**
```latex
\begin{figure}[H]
  \centering
  \includegraphics[width=0.6\textwidth]{apparatus.jpg}
  \caption{Experimental apparatus setup.}
\end{figure}
```

**Fix:** add `\label{fig:apparatus}` immediately after the `\caption{}` line:
```latex
  \caption{Experimental apparatus setup.}
  \label{fig:apparatus}
```

Then in the text (Methods section), add a sentence using `Figure~\ref{fig:apparatus}`.

Compile twice and confirm the figure number in the text matches the figure number in the PDF.
</WorkedExample>

<Activity title="Peer review session" time="30 min" grouping="Pairs" materials={["Laptop", "Partner's Overleaf project link (view-only)", "Printed or shared checklist", "Feedback sheet"]}>
1. Open your partner's Overleaf project. Read the compiled PDF alongside the source code.
2. Work through the 10-item checklist. For every "Fix needed": note the section name and the specific issue on the feedback sheet.
3. Return the feedback sheet to your partner.
4. Begin your own corrections.
</Activity>

<Support title="Correction guide for common checklist failures">
**Check 1 — plain-text formulae still showing:** search (`Ctrl+F`) in the Overleaf source for `O2`, `CO2`, `H2O` etc. Replace each with `\ce{O2}`, `\ce{CO2}`, `\ce{H2O}`.

**Check 3 — figure not cross-referenced:** search for every `\label{fig:...}` in your document. For each one, search the text for `\ref{fig:...}` — if it's missing, add a sentence in the relevant section.

**Check 9 — compile error:** click the red error panel → find the line number → go to that line → fix the syntax. Compile again. Repeat until no errors.

**Check 10 — orphan heading:** look for two consecutive `\section{}` or `\subsection{}` commands with nothing between them. Add at least one sentence.
</Support>
