<Section title="Practical: Access queries and PowerPoint" icon="💻" id="practical">

### Access: AND vs OR — the most important distinction

In Query Design view, criteria placement controls the logic:

| What you want | Where to put the criteria |
|---|---|
| Both conditions must be true (AND) | Same **Criteria** row, in different field columns |
| Either condition can be true (OR) | Different rows — second condition on the **or** row below |

**Example:** Find all employees in "London" who earn more than £30 000.

- AND query: `London` in the City Criteria row AND `>30000` in the Salary Criteria row (same row).
- OR query: `London` in City Criteria row, `>30000` in Salary **or** row (different rows) → returns anyone in London OR anyone earning over £30 000 — a much larger result set.

Run both on the same table and compare. The different result sets make the logic visible.

### Access: other query techniques

- **Wildcard:** `Like "Sm*"` matches Smith, Smyth, Smart, etc.
- **Date criteria:** use hash marks — `#01/01/2024#`.
- **Number criteria:** `>100`, `<=50`, `Between 10 And 20`.
- **Calculated field:** in an empty column header cell, type `NewFieldName: [Field1]*[Field2]`.

Always **run the query** (the red ! button) and check the result set makes logical sense before moving on.

### PowerPoint: the items students consistently miss

#### Slide footer, number and date

Insert → Header and Footer → check the relevant boxes:
- **Date and time** — fixed or automatic as the question specifies.
- **Slide number.**
- **Footer** — type the exact text given in the question.

Click **Apply to All** — not "Apply". This is the most commonly missed step. Clicking "Apply" applies the settings to the current slide only.

#### Transitions

Click the slide in the panel → Transitions tab → choose the transition named in the question. If all slides need the same: click **Apply To All** in the Timing group.

#### Speaker notes

Click the "Click to add notes" area below the slide → type the text exactly as given. Check it appears in Normal view before moving on.

<Section title="Theory: Paper 1 — Systems life cycle, software types & ergonomics" icon="📖" id="theory">

### The systems development life cycle (SDLC)

| Stage | What happens |
|---|---|
| 1. Analysis | Define the problem; find out what the system needs to do |
| 2. Design | Plan the solution — data structures, user interface, logic |
| 3. Development | Build the system (coding, creating databases, etc.) |
| 4. Testing | Test with real/realistic data; errors feed back to Development |
| 5. Implementation | Install and deploy the system for real users |
| 6. Evaluation | Review: does it meet the original requirements? What needs improving? |

**Key exam point:** Testing uses realistic data to find errors; errors discovered in testing loop back to Development. The cycle is iterative, not one-way.

### Software types

| Type | Purpose | Example |
|---|---|---|
| Operating system | Manages hardware; provides user interface; manages files | Windows, macOS, Linux |
| Application software | Performs a specific user task | Word processor, spreadsheet |
| Utility software | Maintains/optimises the system | Antivirus, disk defragmenter, file compression |

**Exam trap:** "Microsoft Word is an operating system." → Word is application software. The operating system manages the hardware and provides the environment in which Word runs.

### Health and ergonomics

| Problem | Cause | Countermeasure |
|---|---|---|
| RSI (repetitive strain injury) | Repeated keyboard/mouse movements | Wrist rests, regular breaks, correct desk height |
| Eye strain | Screen glare, wrong distance or height | Anti-glare screen, correct monitor distance, breaks |
| Back/neck pain | Poor posture, non-adjustable seating | Adjustable chair, lumbar support, monitor at eye level |

<WorkedExample title="Access AND vs OR: checking your result set">
**Task:** "Find all products in the category 'Electronics' with a price less than £50."

AND query setup:
- Category field, Criteria row: `Electronics`
- Price field, Criteria row: `<50`

Both on the **same Criteria row**.

Result check: every record in the results should be an Electronics item AND cost less than £50. If you see any records outside Electronics, or any above £50, something is on the wrong row.

Run the query → count the results → ask "does this number make sense?" If you expected 5 results and got 47, the criteria are probably on different rows (OR logic).
</WorkedExample>

<Support title="Access query checklist">
Before running the query, check:

- [ ] Every required field is added to the grid (double-click the field name).
- [ ] AND conditions: on the **same** Criteria row.
- [ ] OR conditions: on **different** rows.
- [ ] Wildcard uses `Like "text*"` — not just `text*` without `Like`.
- [ ] Text criteria: Access adds quotes automatically — don't type them manually or you get double quotes.
- [ ] Query saved with the exact name specified in the question.
</Support>
