Paper 2 opens with document production in Microsoft Word. The marks test whether you followed the exact sequence — applying a named style is not the same as changing the font to look similar, and a mail merge with the wrong field names earns zero even if the letter looks right. Today you practise each task until the process is reliable.

<Callout type="exam" title="Why this matters" id="why">
  **Paper 2** (2.5 hours) covers document production (Word), databases (Access) and presentations (PowerPoint). Document production accounts for the first section. The most common losses are: applying a font change instead of a named style, omitting field codes from headers/footers, and failing to merge from a data source at all. All three are drilled today.
</Callout>

<KeyTerms terms={[
  { term: "Named style", definition: "A saved set of formatting (font, size, spacing, colour) given a name, e.g. 'Heading 1'. Applying the named style — not just making the text look similar — is what earns the mark." },
  { term: "Field code", definition: "An instruction in a header, footer or mail merge that inserts live data — e.g. the current filename, date, or a merge field from a data source." },

<Section title="Practical: Microsoft Word tasks" icon="💻" id="practical">

### Task 1 — Named styles (most-lost marks)

Home → Styles panel → click the exact style name specified in the question.

**The critical rule:** applying the named style from the Styles panel earns the mark. Changing the font manually to *look* like the style does not — even if the result is visually identical. The mark scheme checks whether the style is applied, not whether it looks right.

To verify: click inside the formatted text → check the Styles panel highlights the correct style name.

### Task 2 — Headers and footers with field codes

Insert → Header (or Footer) → Edit Header, then insert field codes as specified:

| What the question wants | How to insert it |
|---|---|
| Automatic filename | Insert → Quick Parts → Field → choose `FileName` |
| Page number | Insert → Page Number → choose position |
| Date (automatic) | Insert → Quick Parts → Field → choose `Date` |

After inserting: right-click the field → Update Field to confirm it shows correct data. Close Header and Footer when done.

### Task 3 — Mail merge (8 steps)

1. Mailings → Start Mail Merge → **Step-by-Step Mail Merge Wizard**.
2. Select document type (Letters, Labels, etc.).
3. Use the current document as the starting document.
4. Select recipients → Use an existing list → browse to the data file (`.csv` or `.xlsx`).
5. Confirm column headers match what the question asks for — field names are case-sensitive.
6. Insert merge fields: Mailings → Insert Merge Field → select the field name.
7. Preview results — check each merge field shows the correct data.
8. Finish & Merge → Edit Individual Documents → All → OK.

**Common failures:** attaching the wrong data file, inserting fields that don't match the column headers, merging directly to the printer instead of a new document.

### Task 4 — Image text wrapping

Click the image → Picture Tools Format → Wrap Text → choose the wrap type named in the question (Square, Through, In Line with Text, Top and Bottom, etc.).

The question always specifies the wrap type. Choosing the wrong one loses the mark even if the image is positioned correctly.

</Section>

<Section title="Theory: Paper 1 — Safety, security & countermeasures" icon="📖" id="theory">

### Categories of threat

| Category | Examples |
|---|---|
| Malicious external | Virus, hacking, phishing, ransomware |
| Malicious internal | Disgruntled employee leaking data |
| Accidental | User deleting files, hardware failure, natural disaster |

### Logical countermeasures (software/policy)

- **Encryption** — converts data into coded form; only someone with the correct key can read it. Even if intercepted, the data cannot be understood.
- **Passwords** — should be long, complex and unique. Two-factor authentication adds a second check (something you know + something you have).
- **Antivirus** — detects and removes known malware.
- **Firewall** — filters incoming and outgoing network traffic according to rules.
- **Access levels** — different users see different data; a cashier can't access salary records.

### Physical countermeasures (hardware/environment)

- Locked server room (key or keypad).
- Security passes / ID badges for staff.
- CCTV monitoring.
- Biometric entry (fingerprint, retina scan).
- Cable locks on laptops.

### Backups

- **Full backup** — a complete copy of all data. Takes longer; restoring is straightforward.
- **Incremental backup** — only backs up files changed since the last backup. Faster to run; restoring requires the last full backup plus all incrementals.
- **Off-site storage** — essential so a fire or flood doesn't destroy the backup along with the original.

### Phishing vs hacking

- **Phishing** — deceives the user into revealing credentials (fake login pages, urgent emails asking you to "verify your account").
- **Hacking** — exploits a technical vulnerability in a system.

**Exam weak-answer trap:** "Encryption keeps data safe." → *"Encryption converts data into a coded format using an algorithm; only a recipient with the correct decryption key can read it, so intercepted data cannot be understood."*

</Section>

<WorkedExample title="Mail merge: tracing a common failure">
Scenario: A student runs the mail merge wizard, inserts a field called `{First Name}`, previews the results and sees `{First Name}` displayed literally — not the actual name.

**Why it happened:** the data source column is called `FirstName` (no space), but the student inserted a field called `First Name` (with a space). The field names don't match.

**Fix:** in step 4 of the wizard, open the data source and check the exact column headers. Insert the field with the name exactly as it appears in the column header.
</WorkedExample>

<Support title="Mail merge rescue steps">
If the merge didn't produce individual letters:

1. Mailings → Edit Recipient List — confirm the data source is attached.
2. Check field names: Mailings → Insert Merge Field → the dropdown should show your column headers.
3. Preview Results — at least one merged version should show real data.
4. If preview shows field codes literally (e.g. `«First Name»`): the column header doesn't match the inserted field name. Start again from step 4 of the wizard.
</Support>
