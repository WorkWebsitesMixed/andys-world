<Section title="Practical: Excel data analysis" icon="💻" id="practical">

### The function that loses the most marks: VLOOKUP

VLOOKUP has four arguments — get any one wrong and the mark is gone:

```
=VLOOKUP(lookup_value, $table_array$, col_index_num, FALSE)
```

- **lookup_value** — the cell you're looking up (e.g. `A2`).
- **table_array** — the whole lookup table with `$` signs so it stays fixed when copied (e.g. `$E$2:$G$50`). Press **F4** after selecting the range to add `$` automatically.
- **col_index_num** — which column of the table to return (1 = leftmost, 2 = next, etc.).
- **FALSE** — always use `FALSE` for exact match. `TRUE` or omitting it causes wrong results in almost all exam scenarios.

### The other five functions

| Function | Skeleton | Key exam trap |
|---|---|---|
| `IF` | `=IF(condition, value_if_true, value_if_false)` | Nested IF: put the second IF inside value_if_false |
| `COUNTIF` | `=COUNTIF(range, "criteria")` | Range must be a range, not a single cell |
| `SUMIF` | `=SUMIF(range, "criteria", sum_range)` | Three arguments — sum_range is often omitted |
| `AVERAGE` | `=AVERAGE(range)` | Simplest of the six |

### Chart types: match the data to the chart

| Situation | Chart type |
|---|---|
| Comparing separate categories | Bar or Column |
| Showing a trend over time | Line |
| Parts of a whole (percentages) | Pie |
| Two sets of numerical data | XY Scatter |

A "fully labelled" chart needs a **chart title**, **axis titles** for both axes, and **data labels** if specified. These are free marks that proficient students regularly drop.

### Common mark-scheme traps

- VLOOKUP without `FALSE` returns an approximate match — wrong in most exam tasks.
- COUNTIF range written as a single cell (`A2`) instead of a range (`A2:A50`).
- A bar chart chosen when the x-axis is time — use a line chart instead.

</Section>

<Section title="Theory: Paper 1 — Hardware, I/O devices & storage" icon="📖" id="theory">

### Input vs output devices

An **input device** captures data and converts it into a form the computer can process. An **output device** presents processed data to the user.

| Input device | What it captures |
|---|---|
| Keyboard | Keystrokes → text/commands |
| Mouse | Movement + clicks → cursor control |
| Scanner | Physical document → digital file |
| Microphone | Sound → digital audio |
| Barcode reader | Barcode → product code |

| Output device | Output type |
|---|---|
| Monitor | Temporary (soft copy), visual |
| Printer | Permanent (hard copy), physical |
| Speakers | Audio |

Key distinction: **hard copy** (printed, physical) vs **soft copy** (on screen, digital).

### Storage: three types

| Type | Examples | Advantage | Disadvantage |
|---|---|---|---|
| Magnetic | HDD | High capacity, low cost per GB | Moving parts — fragile |
| Optical | DVD, Blu-ray | Portable, cheap to produce | Slower; scratch damage |
| Solid-state | SSD, USB drive | Fast, no moving parts, robust | More expensive per GB |

### RAM vs ROM

- **RAM** — volatile; loses all data when power is removed; holds the OS and running programs.
- **ROM** — non-volatile; keeps data without power; holds the boot instructions the computer needs at startup.

**Classic exam question:** "Why can't the computer boot from RAM alone?" → Because RAM is volatile — it's empty after the power is switched off. ROM is non-volatile so the boot instructions survive.

</Section>

<WorkedExample title="Weak answer → improved answer (storage)">
**Question:** "Explain why solid-state storage is more suitable than a hard disk drive for a laptop."

Weak: *"SSDs are better because they are faster."*

Improved: *"Solid-state drives have no moving parts, making them more robust against physical shock — important in a portable device. They also offer faster data access because there is no read head that needs to seek to the correct track."*

The improvement earns marks by giving **two distinct advantages**, each **linked to the laptop context** with a reason.
</WorkedExample>
