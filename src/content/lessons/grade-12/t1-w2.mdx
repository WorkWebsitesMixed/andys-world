Web authoring is the second half of Paper 3. The marks aren't about design flair — they're about following the exact sequence the examiner expects: external CSS file, alt text on every image, hyperlinks that open the right page. Today you practise each step until the sequence is automatic.

<Callout type="exam" title="Why this matters" id="why">
  **Paper 3** web authoring questions test whether you can produce a functional, correctly structured web page. The most common mark losses are: applying styles inline instead of via an external stylesheet, missing alt text, and hyperlinks that open the wrong target. All three are addressed today.
</Callout>

<Section title="Practical: Expression Web tasks" icon="💻" id="practical">

### Step 1 — the one technique that loses most marks: external CSS

The question will say *"use a stylesheet"*. Applying styles inline (via Format → Font or the toolbar) does **not** earn the mark, even if it looks identical. Follow this sequence every time:

1. **Create the CSS file:** File → New → CSS → name it (e.g. `styles.css`) → Save.
2. **Write a rule:** e.g. `body { background-color: #ffffff; font-family: Arial; }`.
3. **Link it to the HTML page:** Format → CSS → Attach Style Sheet → browse to your `.css` file → OK.
4. **Confirm:** switch to Design view — the style should apply visually.

### Step 2 — images with alt text

1. Insert → Picture → From File → browse to the image → Insert.
2. Right-click the image → Picture Properties → General tab → **Alternate text** field → type a description → OK.

The alt text must **describe the image**, not repeat the filename. "logo.png" earns zero; "School logo — a red shield on a white background" earns the mark.

### Step 3 — hyperlinks

- **Internal (another page in the site):** Select text/image → Insert → Hyperlink (Ctrl+K) → browse to the target `.html` file → OK.
- **External (another website):** Select text/image → Insert → Hyperlink → type the full URL including `https://` → OK.

Always check the link opens the correct target. A link that goes to the wrong page loses the mark even if the hyperlink itself is correctly inserted.

### Step 4 — tables

Insert → Table → Insert Table → set rows and columns → OK. To set column width: right-click the cell → Cell Properties → set width in pixels or %.

### Step 5 — form elements (if required)

Insert → Form → choose element (Text Box, Radio Button, Check Box, Submit Button). Right-click the element → Form Field Properties → set the **Name** field exactly as specified in the question.

### Before finishing: always preview in the browser

File → Preview in Browser (F12). Check: links open the correct page, images display, CSS styles apply correctly in the browser (not just Design view).

</Section>

<Section title="Theory: Paper 1 — Networks, LAN, WAN & protocols" icon="📖" id="theory">

### LAN vs WAN

| | LAN | WAN |
|---|---|---|
| Geographic area | Small — one building or site | Large — cities, countries, global |
| Ownership | Usually one organisation | Multiple organisations; uses public infrastructure |
| Example | School network | The internet |

**Exam sentence:** "A LAN covers a small geographic area and is typically owned and managed by a single organisation. A WAN covers a large geographic area and uses infrastructure owned by multiple organisations."

### Wired vs wireless

| | Wired | Wireless |
|---|---|---|
| Advantage | More reliable, faster, more secure | Mobility, easier to add devices |
| Disadvantage | Cables limit movement | Interference; security risk if unencrypted |

**Scenario question:** "Why might a hospital prefer wired networking?" → Reliability and security are critical; cables provide both.

### Client-server vs peer-to-peer

| | Client-server | Peer-to-peer |
|---|---|---|
| How it works | Clients request; server manages resources | Each device shares equally |
| Advantage | Centralised management, security, backups | Simpler, no single point of failure |
| Best for | Large networks (schools, businesses) | Small home networks |

### Protocols: what they are and why they matter

A **protocol** is a set of rules that governs how data is formatted and transmitted between devices. Without protocols, devices from different manufacturers couldn't communicate.

| Protocol | Purpose |
|---|---|
| HTTP / HTTPS | Transfer of web pages between browser and server (HTTPS adds encryption) |
| FTP | File transfer between computers |
| SMTP | Sending email |
| IMAP / POP3 | Receiving email (IMAP keeps email on server; POP3 downloads it) |

**Weak-answer trap:** "HTTP is used to send emails." → Wrong. HTTP transfers web pages. SMTP sends email.

</Section>

<WorkedExample title="Weak answer → improved answer (protocols)">
**Question:** "A user wants to visit a website. State the protocol used and explain what it does."

Weak: *"HTTP is used. It lets you browse the internet."*

Improved: *"HTTP (HyperText Transfer Protocol) is used. It governs the rules for how web pages are requested by a browser and sent by a web server, defining the format of the request and response messages."*

The improvement earns marks by naming the protocol correctly, **expanding the acronym**, and explaining **what the protocol actually governs** (the request-response format between browser and server).
</WorkedExample>
