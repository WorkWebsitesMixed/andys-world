Four classes. No new firmware. This block is about turning sixty classes of engineering work into something you can explain to someone who has never touched an ESP32.

Students who can only build are engineers in training. Students who can also explain, document, and present are engineers. Block 2D develops three skills that university admissions, internships, and future employers value enormously: technical writing, public speaking, and live demonstration under pressure.

## Class breakdown

| Class | Focus |
|-------|-------|
| 57 | Technical documentation workshop: one-page data sheet + Telegram bot reference card |
| 58 | Presentation rehearsal (two full runs with feedback) + demo room setup |
| 59 | **Public Demo Day** — live event for parents, peers, and administration |
| 60 | Course retrospective · skills inventory · closure |

## What gets presented

The **Crack-the-Code Safe** is the hero project. Earlier builds (RC boat, RoboJam robot, mood lamp) can be displayed at the station as supporting portfolio items.

## The 5-minute presentation structure

Each student presents her safe to visiting guests in 5 minutes:

1. **What it does** — demonstrate the lock, show the OLED UI, show the Telegram bot
2. **How it works** — explain the three-stage puzzle at a level a non-programmer can follow
3. **What was hardest** — one genuine engineering challenge and how it was solved
4. **Take a question** — handle it live, without help

## Documentation deliverables

**One-page data sheet** covering:
- Circuit diagram with all components labelled
- State machine diagram (the 6-state safe flow)
- Component list with quantities and purpose
- Challenge table: stage, difficulty, how to solve it, which IoT feature is used

**Telegram bot reference card** (business-card sized):
- Bot username
- Available commands (`/hint`, `/status`)
- Maximum hint count per round
- Goes on the outside of the safe for Demo Day guests

## Safe-cracking leaderboard

A shared Google Sheet projected in the demo room tracks which guest cracked which safe and how many hints they used. This gives all guests a reason to try all the safes, adds friendly competition, and gives each student real-time feedback on how hard their puzzle actually is.

## Skills inventory (self-assessment, Class 60)

Students fill in what they can now do independently — for their own use, not a grade. Covers C++, electronics, CAD/3D print, IoT protocols, debugging, and communication. Becomes part of their personal portfolio.
